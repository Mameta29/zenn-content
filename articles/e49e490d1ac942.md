## はじめに

JPYC で何かを作ろうとした時に必須になるのは、**チェーンの設定**（Chain ID、RPCさらにはテストネットの情報等）だと思います。

- チェーンの Chain ID・RPC・エクスプローラは？
- テストネットで動作確認するための JPYC とガス代をどこで手に入れる？
- JPYCのアドレスは？

この手の情報はあちこちに散らばっていて集めるのが地味に面倒です。そこでこの記事は、JPYC 開発の最初の一歩で必要になる「**チェーン設定の決定版リファレンス**」を目指します。JPYC コントラクトアドレス・Chain ID・RPC・ブロックエクスプローラ・Faucet を一枚にまとめ、その設定を自分のシステムにどう保持させるかまでを示します。**JPYC で何かを作りたいエンジニア**に向けて、困ったときにここを引けば設定で迷わない、そんな逆引き辞典として使ってもらえたら嬉しいです！

なお、新しいチェーンが JPYC に追加されたら、この記事の「チェーン設定」と「新しいチェーンを追加する」セクションだけ更新すれば、開発の出発点としてずっと使い続けられる ── そういう作りにしてあります。

:::message
RPC URLはときたま変わったりするので、実際に使うときは必ず公式の最新情報をご確認ください。
:::

### 1.3 RPC エンドポイント

**公式パブリック RPC（API キー不要・すぐ試せる）:**

| チェーン | パブリック RPC |
|---|---|
| Ethereum | `https://ethereum-rpc.publicnode.com` |
| Polygon | `https://polygon-rpc.com` |
| Avalanche | `https://api.avax.network/ext/bc/C/rpc` |
| Kaia Mainnet | `https://public-en.node.kaia.io` |
| Sepolia | `https://ethereum-sepolia-rpc.publicnode.com` |
| Polygon Amoy | `https://rpc-amoy.polygon.technology` |
| Avalanche Fuji | `https://api.avax-test.network/ext/bc/C/rpc` |
| Kaia Kairos | `https://public-en-kairos.node.kaia.io` |

動作確認や開発中はこれで十分です。ただしパブリック RPC は共用なので、レート制限が厳しめだったり、混雑時に不安定になったりします。
また、たまにPolygonでは動かなくなったりもするので、こちらの[ChainList](https://chainlist.org/)から都度動いているものを確認して使うこともできます。

**本番運用では商用 RPC を（補足）:**

本番にサービスを出すなら、Alchemy や Infura のような商用 RPC を使いましょう。API キーを取得して、次のような形のエンドポイントを
使います（Kaia は Alchemy / Infura 非対応なので、本番でも公式パブリック RPC か QuickNode などの商用 Kaia RPC を使います）。

| チェーン | Alchemy RPC（`<KEY>` を自分のキーに置換） |
|---|---|
| Ethereum | `https://eth-mainnet.g.alchemy.com/v2/<KEY>` |
| Polygon | `https://polygon-mainnet.g.alchemy.com/v2/<KEY>` |
| Avalanche | `https://avax-mainnet.g.alchemy.com/v2/<KEY>` |
| Sepolia | `https://eth-sepolia.g.alchemy.com/v2/<KEY>` |
| Polygon Amoy | `https://polygon-amoy.g.alchemy.com/v2/<KEY>` |
| Avalanche Fuji | `https://avax-fuji.g.alchemy.com/v2/<KEY>` |

> **本番運用のコツ**: RPC は単一障害点になりやすいです。
> `商用 RPC → 公式パブリック RPC` の順でフォールバックさせるようにしましょう。
> こうしておくと、商用 RPC が一時的に落ちてもパブリック RPC が受け皿になってくれます。Kaia のパブリック RPC を本番で使うならレート制限に注意して、QuickNode などの商用 Kaia RPC を併用するのが安全だと思います。

## 2. Faucet 一覧（テストネットで動かすために）

### 2.1 JPYC Faucet（テスト用 JPYC の入手）

JPYC 公式が **テストネット用 JPYC の配布ツール**を提供してくれています。

**https://faucet.jpyc.co.jp**

ここでウォレット接続をして、対応テストネット（Sepolia / Amoy / Fuji / Kairos など）を選ぶと、テスト用 JPYC を取得できます。配布上限や1日1回制限などあるので、サイト側の表示に従ってください。

### 2.2 ネイティブガストークンの Faucet

ガス代になるネイティブトークンは、チェーンごとに別の Faucet から入手します。

| テストネット | ガス | Faucet |
|---|---|---|
| Sepolia | ETH | https://www.alchemy.com/faucets/ethereum-sepolia ／ https://sepolia-faucet.pk910.de/ |
| Polygon Amoy | POL | https://www.alchemy.com/faucets/polygon-amoy ／ https://faucets.chain.link/polygon-amoy |
| Avalanche Fuji | AVAX | https://core.app/tools/testnet-faucet/ ／ https://faucets.chain.link/fuji |
| Kaia Kairos | KAIA | https://faucet.kaia.io |

> ただし、最近では多くのFaucetがメインネットの少額のPOLやETHが必要といった制限があったりします。上記の中で制限なく取得できるのは今のところ、Sepoliaの https://sepolia-faucet.pk910.de/ とKaia Kairosの https://www.kaia.io/faucet のみです。

## 3. チェーン設定をシステムに組み込む

ここからは少し実装の話です。上記で示したチェーン設定を、**自分のシステムにどう保持させるか**を見ていきます。スタックは viem / TypeScript を想定しています。「JPYC で何をするか」には踏み込まず、その手前のチェーンを正しく設定するところまでです。

### 3.1 「チェーンレジストリ」を1ファイルにまとめる

アドレスや Chain ID は**1つの設定オブジェクト**に集約しましょう。

```typescript
// config/chains.ts
export interface ChainConfig {
  id: number
  name: string
  isTestnet: boolean
  jpycContractAddress: `0x${string}`
  rpcUrls: readonly string[]   // フォールバック用に複数持つ
  nativeTokenSymbol: string
  explorerUrl: string
}

// JPYC は全チェーン同一アドレス（§1.1）。定数化しておく。
const JPYC = '0xE7C3D8C9a439feDe00D2600032D5dB0Be71C3c29' as const

export const CHAINS: Record<number, ChainConfig> = {
  // ---- Mainnets ----
  1: {
    id: 1, name: 'Ethereum Mainnet', isTestnet: false,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://eth-mainnet.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'ETH', explorerUrl: 'https://etherscan.io',
  },
  137: {
    id: 137, name: 'Polygon Mainnet', isTestnet: false,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://polygon-mainnet.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'POL', explorerUrl: 'https://polygonscan.com',
  },
  43114: {
    id: 43114, name: 'Avalanche C-Chain', isTestnet: false,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://avax-mainnet.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'AVAX', explorerUrl: 'https://snowtrace.io',
  },
  8217: {
    id: 8217, name: 'Kaia Mainnet', isTestnet: false,
    jpycContractAddress: JPYC,
    rpcUrls: ['https://public-en.node.kaia.io'],
    nativeTokenSymbol: 'KAIA', explorerUrl: 'https://kaiascan.io',
  },
  // ---- Testnets ----
  11155111: {
    id: 11155111, name: 'Sepolia', isTestnet: true,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://eth-sepolia.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'ETH', explorerUrl: 'https://sepolia.etherscan.io',
  },
  80002: {
    id: 80002, name: 'Polygon Amoy', isTestnet: true,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://polygon-amoy.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'POL', explorerUrl: 'https://amoy.polygonscan.com',
  },
  43113: {
    id: 43113, name: 'Avalanche Fuji', isTestnet: true,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://avax-fuji.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'AVAX', explorerUrl: 'https://testnet.snowtrace.io',
  },
  1001: {
    id: 1001, name: 'Kaia Kairos', isTestnet: true,
    jpycContractAddress: JPYC,
    rpcUrls: ['https://public-en-kairos.node.kaia.io'],
    nativeTokenSymbol: 'KAIA', explorerUrl: 'https://kairos.kaiascan.io',
  },
}

export function getChainConfig(chainId: number): ChainConfig {
  const c = CHAINS[chainId]
  if (!c) throw new Error(`Unsupported chain ID: ${chainId}`)
  return c
}

/** JPYC アドレス取得は必ずこの関数経由にする（直書き禁止） */
export function getJpycAddress(chainId: number): `0x${string}` {
  return getChainConfig(chainId).jpycContractAddress
}

/** エクスプローラの tx URL を組み立てる */
export function getExplorerTxUrl(chainId: number, txHash: string): string {
  return `${getChainConfig(chainId).explorerUrl}/tx/${txHash}`
}
```

これで「JPYC アドレスもエクスプローラ URL も、この1ファイルの関数からしか取らない」という規律が敷けます。チェーンが増えても触るのはここだけ。上記のようにしておくことで、JPYC対応のチェーンが増えても下記の部分を新しく追加するだけですみます。
```
1: {
    id: 1, name: 'Ethereum Mainnet', isTestnet: false,
    jpycContractAddress: JPYC,
    rpcUrls: [`https://eth-mainnet.g.alchemy.com/v2/${process.env.ALCHEMY_API_KEY}`],
    nativeTokenSymbol: 'ETH', explorerUrl: 'https://etherscan.io',
}
```

## まとめ

今回は、JPYC で何かを作りたい開発者が最初に必ず必要になる「チェーン設定」をテーマに、チェーン設定・Faucet 一覧・設定の組み込み方を 1 つのリファレンスにまとめてみました。


JPYC は規制対応のステーブルコインとして対応チェーンを着実に増やしていて、これからも新しいチェーンが加わっていくはずです。そのたびにこの記事も更新していくので、**困ったらこのページをブックマークして引いてもらえれば**、JPYC 開発の出発点としてずっと使えるようにしておきます。

JPYC の関数や決済ユースケースの実装については別記事で解説しているので、設定が終わったらぜひそちらも覗いてみてください。
